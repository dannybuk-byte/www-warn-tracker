## CRITICAL JS CONSTRAINTS (iframe sandbox — never violate)

- **Zero backticks** — no template literals. `js.count('`') % 2 == 0`
- **No `\u{XXXX}` escapes** — use literal emoji or `\uXXXX` (4-digit only)
- **No `?.` optional chaining** — use `&&` guards
- **No `=>` arrow functions** — use `function(x) {}`
- **No `const`/`let`** — `var` only
- **No split `.then()` across lines** (ASI bug) — keep on same line as `})`
- **No `href="#id"`** — use `onclick="document.getElementById('id').scrollIntoView({behavior:'smooth'})"`
- **No `<form>` tags**, no `localStorage`
- **CSS stays in `<style>` block** — never inside `<script>`
- **Large file edits** — use single Python script (read → all replacements in memory → write once)

## Demo Pipeline Flow

### Live Scrape mode (`_demoMode = 'live'`):
1. `_runLiveScrape()` → connects DOL, step 1 done
2. `_streamFilings(idx, ot)` → appends `<tr id="wrow-N" class="warn-row-live">` as skeleton rows one by one (~350ms each), counter updates
3. `_runSecAndCeo(ot)` → logs SEC contradictions per company in terminal
4. `_resolveAllRows(ot)` → updates each skeleton row to final state; EVADING rows get `flash-red` CSS animation

### Batch mode (`_demoMode = 'batch'`):
1. `_runBatchAnalysis()` → original step-by-step terminal animation
2. `_finishBatch()` → builds full results tbody at once via `innerHTML`

**Row ID pattern (live mode only):** `wrow-0` through `wrow-9` (index into `_warnData`).

## Outreach Composer Flow

1. User drills into a dossier in Live Demo
2. Clicks **✍️ Compose Outreach** button at bottom of dossier card
3. `openComposer(worker)` called with full worker object → switches to compose tab, pre-fills worker card, auto-selects best platform, updates 60-day window
4. User selects platform + tone + optional custom context
5. Clicks **✨ Generate Message** → `composeMessage()` builds prompt and calls Claude API (`claude-sonnet-4-20250514`)
6. Rendered to `#compose-output`; subject line extracted for `work_email` platform
7. **⎘ Copy** or **+ Add to Queue** → `_composerQueue[]` / `_renderComposerQueue()`

**Composer state vars:** `_composerWorker`, `_composerCompany`, `_composerPlatform`, `_composerTone`, `_composerQueue[]`, `_composerOutput`, `_composerSubject`

**Fallback:** `_fallbackMessage(p, platform, tone, daysLeft, company, firstName)` — hardcoded templates for all 5 platforms, fires if API unavailable.
